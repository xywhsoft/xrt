# XRT 范例程序完整规划 (Examples Specification)

---

## 一、通用标准

### 1.1 目录结构

```
examples/
├── bin/                    <- 统一编译输出目录
├── build_all.bat           <- Windows 一键编译所有范例
├── build_all.sh            <- Linux 一键编译所有范例
└── {子库名}/
    └── {范例名}/
        └── main.c
```

### 1.2 引用方式

所有 main.c 统一使用 singlehead 单头文件：

```c
#define XRT_IMPLEMENTATION
#include "../../../singlehead/xrt.h"
```

### 1.3 统一编译输出

- 所有范例编译输出到 `examples/bin/` 目录
- 可执行文件命名规则：`{子库名}_{范例名}.exe` (Windows) / `{子库名}_{范例名}` (Linux)
- 例如：`base_memory_basic.exe`、`string_split.exe`、`tools_chatroom.exe`
- `build_all.bat` / `build_all.sh` 为一键编译脚本
- **严禁在编译脚本中使用 pause、cmd /k 等阻塞命令**，确保全程无人值守执行
- 编译脚本出错时仅打印错误信息并继续编译下一个，不中断整体流程

### 1.4 统一顶部注释模板

```c
/*
 * XRT Example - {范例名称 (英文)}
 * XRT 范例 - {范例名称 (中文)}
 *
 * Description / 说明:
 *   EN: {英文功能描述}
 *   CN: {中文功能描述}
 *
 * Build / 编译:
 *   tcc main.c -o ../../bin/{输出名}.exe          (Windows, TCC)
 *   gcc main.c -o ../../bin/{输出名} -lm           (Linux, GCC)
 *
 * Note / 注意:
 *   - {注意事项1}
 *   - {注意事项2}
 */
```

### 1.5 编码规范

- 使用 Tab 缩进
- 匈牙利命名法 (整数 i, 浮点 f, 字符串 s, 指针 p, 数组 arr, 布尔 b 等)
- 函数花括号单独一行，if/for/while 花括号同行
- 小括号内外空格：`if ( condition ) {`
- 组合条件加括号：`if ( (a == 1) && (b == 2) ) {`
- 中英双语注释 (英文在上，中文在下)
- Linux 编译时需要 `-lm` (math)、线程相关需 `-lpthread`、网络需 `-lws2_32` (Windows)

### 1.6 代码结构统一模式

```c
int main()
{
    // Initialize XRT library
    // 初始化 XRT 库
    xrtInit();

    // ... example code ...

    // Cleanup XRT library
    // 清理 XRT 库
    xrtUnit();
    return 0;
}
```

## 二、范例程序清单

### 2.1 base - 基础模块 (4 个范例)

| # | 工程名 | 路径 | 功能说明 |
|---|--------|------|----------|
| 1 | `init_and_config` | `examples/base/init_and_config/` | xrtInit/xrtUnit 初始化和销毁流程，展示 xCore 全局数据结构各字段 |
| 2 | `memory_basic` | `examples/base/memory_basic/` | xrtMalloc/xrtCalloc/xrtRealloc/xrtFree 基础内存管理操作 |
| 3 | `temp_memory` | `examples/base/temp_memory/` | xrtTempMemory 环形临时内存机制，演示自动释放和 32 槽位循环特性 |
| 4 | `error_handling` | `examples/base/error_handling/` | xrtSetError/xrtClearError/OnError 回调的错误处理机制 |

**实现要点：**
- `init_and_config`：打印 xCore.AppFile, AppPath, sNull 等字段；展示自定义内存分配器替换
- `memory_basic`：分配不同大小内存、realloc 扩容、calloc 零初始化验证
- `temp_memory`：连续申请 35 次临时内存验证环形回收，验证第 33 次会回收第 1 次
- `error_handling`：注册 OnError 回调函数，故意触发错误（如分配 0 字节），检查 xCore.LastError

## 四、实施计划

### 4.1 实施方式

- **逐个实现**：一次完成一个范例项目，立即继续下一个，不停顿不询问
- **无人值守**：全程自动执行，不弹出确认、不 pause、不等待输入
- **统一验证**：所有范例完成后，通过 build_all.bat/sh 一键编译到 examples/bin/ 进行统一验证
- **错误处理**：单个范例编译失败不影响其他范例，脚本继续执行并记录失败信息

### 4.2 执行顺序

按照依赖关系从底层到上层的顺序逐个实现：

**Phase 1 - 基础层** (约 19 个)：base -> math -> string -> charset -> path -> jnum

**Phase 2 - 系统层** (约 14 个)：os -> time -> file -> hash

**Phase 3 - 数据结构层** (约 18 个)：buffer -> array -> stack -> bsmm -> memunit -> mempool

**Phase 4 - 高级数据层** (约 15 个)：avltree -> dict -> list -> value

**Phase 5 - 处理引擎层** (约 10 个)：json -> template

**Phase 6 - 并发层** (约 10 个)：thread -> coroutine

**Phase 7 - 网络与安全层** (约 24 个)：crypto -> network -> http -> xid

**Phase 8 - 经典工具层** (10 个)：tools (依赖前面所有模块的经验，最后实现)

每个 Phase 内按表中顺序从上到下实现。
