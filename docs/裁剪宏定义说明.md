# XRT 模块裁剪说明

> 当前源码主线的模块裁剪指南。本文档只描述当前仍然有效的裁剪宏，不再保留旧网络模块的历史裁剪面。

[返回索引](README.md)

---

## 1. 目标

XRT 支持通过预处理宏按需裁剪模块，以达到：

- 减小编译体积
- 降低依赖面
- 只保留需要的功能
- 继续保持单头文件与轻量化特性

裁剪原则：

- 裁剪必须围绕**当前正式主线**
- 已淘汰模块的历史裁剪宏不再作为正式配置面保留


## 2. 当前有效裁剪宏

### 2.1 运行时与基础模块

```c
#define XRT_NO_TIME
#define XRT_NO_FILE
#define XRT_NO_THREAD
#define XRT_NO_COROUTINE
#define XRT_NO_CRYPTO
#define XRT_NO_XID
#define XRT_NO_BUFFER
#define XRT_NO_REGEX
```

### 2.2 网络主线模块

```c
#define XRT_NO_NETWORK
#define XRT_NO_NETTLS
#define XRT_NO_XURL
#define XRT_NO_HTTP_UTIL
#define XRT_NO_XCODEC
#define XRT_NO_XHTTP
#define XRT_NO_XHTTPD
#define XRT_NO_XWS
```

说明：

- `XRT_NO_NETWORK` 会整体关闭当前网络主线
- 如果不关闭 `XRT_NO_NETWORK`，仍可继续按子模块细分裁剪

### 2.3 内存与数据结构模块

```c
#define XRT_NO_ARRAY
#define XRT_NO_BSMN
#define XRT_NO_MEMUNIT
#define XRT_NO_MEMPOOL_FS
#define XRT_NO_STACK
#define XRT_NO_AVLTREE
#define XRT_NO_MEMPOOL
#define XRT_NO_DICT
#define XRT_NO_LIST
```

### 2.4 高层数据与文本模块

```c
#define XRT_NO_VALUE
#define XRT_NO_JNUM
#define XRT_NO_JSON
#define XRT_NO_TEMPLATE
```


## 3. 当前已经淘汰的历史裁剪面

以下历史裁剪宏已不再属于当前正式主线：

```c
XRT_NO_NETSOCK
XRT_NO_NETPOLL
XRT_NO_NETLOOP
XRT_NO_NETTCP
XRT_NO_NETUDP
XRT_NO_NETHTTP
```

原因：

- 旧网络模块族已不再是当前正式网络主线
- 当前正式网络主线是：
	- `xnet-v2`
	- `xtlssession`
	- `xhttp / xhttpd / xws`


## 4. 自动依赖关系

当前源码中仍保留自动依赖收口逻辑。

### 4.1 Template

启用模板引擎时，自动启用：

- `VALUE`
- `JNUM`
- `DICT`
- `LIST`
- `AVLTREE`
- `BSMN`
- `MEMUNIT`
- `MEMPOOL_FS`

### 4.2 JSON

启用 JSON 时，自动启用：

- `VALUE`
- `DICT`
- `LIST`
- `AVLTREE`
- `BSMN`
- `MEMUNIT`
- `MEMPOOL_FS`

### 4.3 VALUE

启用 VALUE 时，自动启用：

- `DICT`
- `LIST`
- `AVLTREE`
- `BSMN`
- `MEMUNIT`
- `MEMPOOL_FS`

### 4.4 MemPool / FSMemPool

启用：

- `MEMPOOL`
- `MEMPOOL_FS`

时，自动启用：

- `BSMN`
- `MEMUNIT`

### 4.5 Dict / List

启用：

- `DICT`
- `LIST`

时，自动启用：

- `AVLTREE`


## 5. 最小配置

当前源码仍支持：

```c
#define XRT_MINIMAL
```

它用于快速得到一个极简运行时起点。

注意：

- `XRT_MINIMAL` 只是基础裁剪起点
- 如果后续显式启用了高层模块，相关依赖会被重新拉起


## 6. 推荐配置方式

### 6.1 源码树中的最小运行时

在当前源码树主线中，通常直接在主 `main.c` 中配置裁剪宏：

```c
#define XRT_MINIMAL
#include "xrt.h"
```

适用：

- 小工具
- 文本处理
- 基础脚本式程序

说明：

- 当前主线优先以源码树构建为基线
- 单头文件分发仍然支持同样的裁剪宏，但不再作为文档中的默认主线讲解方式

### 6.2 不含网络的运行时

```c
#define XRT_NO_NETWORK
#include "xrt.h"
```

适用：

- 只使用运行时、容器、JSON、模板、正则
- 不需要网络主线

### 6.3 保留网络主线但关闭应用层

```c
#define XRT_NO_XHTTP
#define XRT_NO_XHTTPD
#define XRT_NO_XWS
#include "xrt.h"
```

适用：

- 只需要底层网络、TLS、wait-source、future bridge
- 不需要 HTTP client/server 与 WebSocket

### 6.4 关闭 TLS

```c
#define XRT_NO_NETTLS
#include "xrt.h"
```

适用：

- 明确只做明文网络通信
- 不需要 `xtlssession`


## 7. 使用建议

### 7.1 优先从主线模块角度裁剪

推荐思路：

- 先决定是否需要网络主线
- 再决定是否需要高层数据模块
- 最后裁剪内存与容器层

不要继续围绕旧网络模块族的思路配置。

### 7.2 裁剪后必须重新验证

每次裁剪配置变化后，应至少验证：

- `xrt.c` 可编译
- 主测试程序可编译
- 当前项目需要的入口模块仍可工作

### 7.3 裁剪不等于绕过依赖

如果显式启用了高层模块，XRT 会按当前依赖关系自动恢复必须的基础模块。

这不是冗余，而是为了保证：

- 单头文件模式稳定
- 模块依赖不被错误打断


## 8. 当前不建议保留的历史写法

以下写法不应继续出现在新文档、新示例和新项目模板中：

```c
#define XRT_NO_NETTCP
#define XRT_NO_NETUDP
#define XRT_NO_NETHTTP
```

它们已经不代表当前正式网络主线。


## 9. 验收标准

当前裁剪文档必须满足：

- 只出现当前有效裁剪宏
- 不再把旧网络模块裁剪面当正式入口
- 能准确反映 `xrt.h / xrt.c` 中的真实依赖链
- 与当前网络、TLS、应用层主线保持一致
